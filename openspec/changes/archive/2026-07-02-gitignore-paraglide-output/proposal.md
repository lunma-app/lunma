## Why

Plumbing (shape b). Consumer: CLAUDE.md's own **Quality gates** policy — `pnpm verify` must be green and the working tree trustworthy before every commit, for every contributor and every future change that touches i18n. That trust is currently broken: `apps/extension/src/shared/paraglide/**` is committed (per D2 in `openspec/changes/archive/2026-06-28-add-i18n-foundation/design.md`), but it has two independent producers — the standalone `paraglide-js compile` CLI (`gen:i18n`, run before `vite`/`vite build`) and `paraglideVitePlugin` (configured in `vite.config.ts`, which also self-compiles on its own `buildStart` hook). Running both against the same output directory is not a pattern used by any official `@inlang/paraglide-js` example (Vite, TanStack Start, Astro all configure the Vite plugin alone), and their independently-versioned, environment-dependent defaults have already drifted from each other and from the committed baseline three separate times this week: the per-message file layout silently flipping between `message-modules`/`locale-modules` depending on `NODE_ENV` (so `vitest run`/`svelte-check` rewrote the entire message directory on every `pnpm verify`), the Vite plugin unconditionally overriding `isServer` against the CLI's default, and the CLI baking a machine-dependent absolute path into the generated README. Each incident required someone to notice a "dirty" working tree, diagnose which of the two compilers was right, and hand-resync them — a fully generated-and-gitignored output makes the entire incident class structurally impossible, and matches upstream's own default (`emitGitIgnore` defaults to `true` in `@inlang/paraglide-js`, which Lunma currently overrides to `false`).

## What Changes

- **Reverse D2**: stop committing `apps/extension/src/shared/paraglide/**`. Gitignore it at the repo root, following the existing `derived-controls.generated.ts` precedent (a comment explaining what regenerates it and when). Untrack the currently-committed files (`git rm -r --cached`); they remain on disk, regenerated on demand.
- **Drop the redundant `gen:i18n` CLI prestep** from `apps/extension/package.json`'s `dev` and `build` scripts. `paraglideVitePlugin` (already in `vite.config.ts`) fully self-compiles via its own `buildStart` hook — confirmed empirically (`vite build` alone, output directory deleted beforehand, fully regenerated every file with zero prestep). Running the CLI immediately before it added a second, differently-configured compiler for no benefit.
- **Add a `postinstall` script** to `apps/extension/package.json` (`pnpm gen:i18n`) so a bare `pnpm install` leaves `typecheck` (`tsc --noEmit`) and `check` (`svelte-check`) — the only two scripts that run outside Vite and can't self-heal — working immediately on a clean checkout or in CI, without needing `gen:i18n` prepended to each of those scripts individually. (`test:run`, `lint`, `lint:styles`, and every catalog script are unaffected either way: vitest self-heals via the same Vite plugin, and the others never touch or need the paraglide directory.)
- **Harden the two remaining producers against future drift**: pass explicit, matching `--output-structure`/`isServer` values to both `gen:i18n`'s CLI flags and `paraglideVitePlugin`'s config, instead of relying on each tool's own independently-versioned defaults happening to agree. This is no longer needed to prevent git noise (nothing is committed), but keeps the on-disk shape stable across `postinstall` vs. `dev`/`build`/`test` and future-proofs against an upstream `@inlang/paraglide-js` version bump silently changing either side's default again.
- **Remove the paraglide-specific `linguist-generated`/`-diff` lines from `.gitattributes`** (moot once nothing is committed). The `icon-loaders.generated.ts` line is untouched — that generator has a single deterministic producer (one Node script, no second bundler-plugin re-touching its output) and doesn't share this risk profile, so it stays committed as-is.
- **Docs updated in the same change** (per CLAUDE.md's deviation/doc-lockstep policy): `docs/tech-stack.md` (two mentions of "committed"), `docs/architecture.md` (the source tree diagram plus two prose mentions), and the `i18n` capability spec (`openspec/specs/i18n/spec.md`, the "Messages SHALL be consumed only through..." requirement). Left untouched: the import-DAG / layer-boundary docs, since the generated runtime still lives under `shared/` and still imports nothing from other layers — that part of D2 is unaffected.

## Capabilities

### New Capabilities
(none)

### Modified Capabilities
- `i18n`: the "Messages SHALL be consumed only through the compiled Paraglide runtime..." requirement changes from "regenerated by a `gen:i18n` script... and committed to the repository" to "gitignored and regenerated on demand — via `paraglideVitePlugin` for `dev`/`build`/`test`, and a `postinstall` hook for a cold `typecheck`/`check`".

## Impact

- **`apps/extension/package.json`**: `dev`/`build` scripts (drop the `gen:i18n` prestep), `gen:i18n` script (add explicit `--output-structure`/`--is-server` flags), new `postinstall` script.
- **`apps/extension/vite.config.ts`**: `paraglideVitePlugin`'s `isServer`/`outputStructure` options pinned explicitly (already applied in the working tree during investigation; carried forward and finalized by this change).
- **`.gitignore`**: new entry for `apps/extension/src/shared/paraglide/**`.
- **`.gitattributes`**: remove the paraglide-specific `linguist-generated`/`-diff` line.
- **`apps/extension/src/shared/paraglide/**`**: untracked from git (`git rm -r --cached`); stays on disk as build output.
- **`docs/tech-stack.md`**, **`docs/architecture.md`**: prose + tree-diagram updates.
- **`openspec/specs/i18n/spec.md`**: requirement text updated.
- **CI (`.github/workflows/ci.yml`)**: no change anticipated — `pnpm install --frozen-lockfile` already runs before each of the 5 extension matrix jobs, and pnpm runs a workspace package's `postinstall` automatically as part of that install; to be confirmed empirically during implementation.
- No new public types, files, methods, or exported symbols — this is a build-tooling/process-only change with no user-visible surface, so the component-library primitive rules don't apply.
