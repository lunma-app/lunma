## MODIFIED Requirements

### Requirement: Supported locales and the message-catalog contract

Lunma SHALL define a fixed set of supported UI locales: `en` (the base locale and
source of truth) plus `es`, `pt`, `fr`, `de`, `ja`, `ko`, `zh-CN`, `ru`. The set
SHALL be declared once in `apps/extension/project.inlang/settings.json`
(`baseLocale: "en"`, `locales: [...]`) and mirrored by the `SupportedLocale` union type
exported from `apps/extension/src/shared/settings.ts`. The Portuguese locale is
region-neutral `pt` (not `pt-PT`): a single Portuguese catalog serves every
Portuguese browser, whether European or Brazilian.

UI message strings SHALL live in per-locale catalog files
`apps/extension/messages/{locale}.json` (one file per supported locale), outside
`apps/extension/src/` so they are excluded from the TypeScript, Biome, and Stylelint
globs. `en.json` SHALL be the authoritative source; until a downstream change supplies
translations, the other locale files MAY contain English copies, but every file SHALL
carry the same key set (see the parity requirement).

Messages SHALL be consumed only through the compiled Paraglide runtime generated into
`apps/extension/src/shared/paraglide/`. The generated directory SHALL be gitignored,
not committed, and regenerated on demand: `paraglideVitePlugin` (configured in
`vite.config.ts`) SHALL regenerate it automatically for `dev`/`build`/`vitest`, and a
`postinstall` script (`gen:i18n`, wrapping `paraglide-js compile`) SHALL regenerate it
once after `pnpm install` so a cold `typecheck`/`check` (the two scripts that run
outside Vite and cannot self-heal) work without a manually-remembered extra step. The
generated directory SHALL be excluded from Biome linting. Because it lives under
`shared/`, the generated runtime SHALL import nothing from other `apps/extension/src/`
layers, keeping the import DAG legal.

#### Scenario: The locale set is single-sourced

- **WHEN** the supported-locale set is needed by the inlang project and by the `SupportedLocale` type
- **THEN** both SHALL list exactly `en, es, pt, fr, de, ja, ko, zh-CN, ru` with `en` as the base

#### Scenario: Generated runtime keeps the layer DAG legal

- **WHEN** `biome check` runs over `apps/extension/src/shared/paraglide/`-consuming code
- **THEN** no import from `ui`, `background`, `sidebar`, `launcher`, `options`, or `content` into `shared/` SHALL be introduced by the i18n runtime, and `biome check` SHALL pass

#### Scenario: A clean checkout typechecks without a manual generation step

- **GIVEN** a fresh clone with no `apps/extension/src/shared/paraglide/` directory on disk
- **WHEN** `pnpm install` completes
- **THEN** its `postinstall` step SHALL have regenerated `apps/extension/src/shared/paraglide/`, so `pnpm typecheck` and `pnpm check` SHALL both pass with no further manual step

#### Scenario: The generated runtime is never committed

- **WHEN** `apps/extension/src/shared/paraglide/` is regenerated by any of its producers (the Vite plugin or the `postinstall` CLI compile)
- **THEN** `git status` SHALL report no changes under that directory, because it is excluded via the root `.gitignore`
